# AI_Agent_Artik

Персональный AI-агент на Node.js. Проект умеет работать как консольный ассистент и как Telegram-бот, хранит память по чатам, переключает роли, вызывает инструменты и анализирует результаты через LLM.

## Что умеет агент

- Отвечает на обычные вопросы через OpenRouter.
- Работает в нескольких ролях: обычный агент, JavaScript-наставник и архитектор.
- Запоминает пользовательские факты в долговременную память.
- Поддерживает хранение долговременной памяти в PostgreSQL через `DATABASE_URL`.
- Хранит краткосрочный контекст текущего диалога.
- Выполняет простые команды: время, калькулятор, UUID, случайное число, Base64.
- Получает погоду через `wttr.in`.
- Ищет информацию через Tavily.
- Ищет видео на YouTube через YouTube Data API.
- Читает веб-страницы по URL и отдает текст страницы в анализатор.
- Получает новости через Google News RSS.
- Анализирует кодовую базу проекта через `Codebase Analyzer`.
- Работает в Telegram, включая текстовые сообщения, voice и audio.
- Делит длинные ответы для Telegram на части.

## Быстрый старт

1. Установи зависимости:

```bash
npm install
```

2. Создай `.env` в корне проекта.

Минимальный вариант для обычного AI-чата:

```env
OPENROUTER_API_KEY=твой_ключ_openrouter
MODEL_DEFAULT=openai/gpt-4o-mini
```

Опционально подключи PostgreSQL для долговременной памяти:

```env
DATABASE_URL=postgresql://user:password@localhost:5432/ai_agent_artik
```

Для облачных PostgreSQL-провайдеров, которым нужен SSL, добавь:

```env
DATABASE_SSL=require
```

3. Запусти CLI-режим:

```bash
npm start
```

4. Запусти Telegram-бота:

```bash
npm run telegram
```

5. Запусти тесты:

```bash
npm test
```

| Переменная | Для чего нужна |
| --- | --- |
| `OPENROUTER_API_KEY` | Обязательна для обычных ответов AI, роутера инструментов, анализа результатов и распознавания аудио. |
| `MODEL_DEFAULT` | Основная модель для агента. |
| `OPENROUTER_MODEL` | Запасная основная модель, если `MODEL_DEFAULT` не задана. |

| `MODEL_ARCHITECT` | Модель для режима `architect`. Необязательно. |

| `TELEGRAM_BOT_TOKEN` | Токен Telegram-бота от BotFather. Нужен для `npm run telegram`. |
| `TAVILY_API_KEY` | Нужен для интернет-поиска через `/search` и похожие запросы. |
| `YOUTUBE_API_KEY` | Нужен для поиска YouTube-видео. |
| `OPENROUTER_STT_MODEL` | Модель распознавания речи. По умолчанию используется `openai/whisper-large-v3`. |
| `MODEL_STT` | Альтернативное имя переменной для STT-модели. |
| `OPENROUTER_STT_LANGUAGE` | Язык распознавания речи. |
| `STT_LANGUAGE` | Альтернативное имя переменной языка STT. |
| `DATABASE_URL` | Строка подключения PostgreSQL. Если задана, долговременная память хранится в таблице `memories`. |
| `DATABASE_SSL` | SSL-режим PostgreSQL: `require`/`true` включает SSL, `disable`/`false` выключает. |

## Роли агента

### `default`

Обычный персональный режим. Используется для общих вопросов, советов, объяснений, технических задач и повседневного общения.

### `coder`

Режим Senior JavaScript Developer и AI Engineer. Подходит для:

- JavaScript и Node.js;
- Telegram-ботов на Telegraf;
- AI-агентов;
- объяснения кода;
- готовых структур проектов.

Включить режим:

```text
/agent coder
```

Или разово:

```text
/coder объясни Promise.allSettled
```

### `architect`

Режим Solution Architect и AI Architect. Подходит для:

- проектирования систем;
- выбора стека;
- архитектурных ревью;
- планов разработки;
- масштабируемости и поддержки.

Включить режим:

```text
/agent architect
```

Или разово:

```text
/architect спроектируй AI-агента для Telegram
```

## Память

Долговременная память хранится в PostgreSQL, если в `.env` задана переменная `DATABASE_URL`. При старте агент создаёт таблицу `memories`, если её ещё нет. Если `DATABASE_URL` не задана, агент автоматически использует прежний файловый fallback `memory.json` в корне проекта.

Память разделяется по `chatId`, поэтому разные Telegram-чаты получают разные наборы записей.

Примеры:

```text
запомни я учу JavaScript
память
/history
/forget 1
/clear
```

Краткосрочная история хранится в объекте агента во время работы процесса. Она помогает модели помнить последние сообщения диалога. Лимит истории: последние 100 сообщений.

## Инструменты

### `time`

Возвращает локальное время через `new Date().toLocaleString()`.

### `calc`

Считает только простые математические выражения. Разрешены цифры, пробелы, скобки и операторы `+ - * / %`.

Пример:

```text
calc 2 + 2 * 3
```

### `weather`

Получает короткий прогноз через `wttr.in`.

Пример:

```text
/weather Луцк
```

### `search`

Ищет информацию через Tavily API. Нужен `TAVILY_API_KEY`.

Пример:

```text
/search что такое Model Context Protocol
```

### `webReader`

Открывает URL, удаляет `script` и `style`, достает текст из `body`, сжимает пробелы и возвращает первые 12000 символов.

Пример:

```text
прочитай сайт https://nodejs.org
```

### `youtube`

Ищет до 5 видео через YouTube Data API. Нужен `YOUTUBE_API_KEY`.

Пример:

```text
/youtube JavaScript event loop
```

### `news`

Берет новости из Google News RSS.

Пример:

```text
/news искусственный интеллект
```

### `codebase`

Собирает snapshot проекта и отправляет его в `resultAnalyzer`.

В snapshot попадают:

- ключевые файлы корня: `index.js`, `package.json`, `README.md`, `jsconfig.json`, `eslint.config.js`;
- директории `src` и `test`;
- файлы `.js`, `.ts`, `.tsx`, `.jsx`, `.mjs`, `.cjs`, `.json`, `.md`;
- дерево файлов;
- содержимое файлов с лимитами.

Исключаются:

- `node_modules`;
- `.git`;
- `dist`;
- `build`;
- `coverage`;
- `.next`;
- `tmp`.

Лимиты:

- до 8000 символов на файл;
- до 70000 символов всего.

Ответ Codebase Analyzer строится в формате:

1. Краткий вывод
2. Архитектура проекта
3. Сильные стороны
4. Проблемы
5. Что исправить первым
6. Рекомендации по файлам

## Telegram-бот

Telegram-режим запускается командой:

```bash
npm run telegram
```

Нужна переменная:

```env
TELEGRAM_BOT_TOKEN=твой_telegram_bot_token
```

Бот умеет:

- отвечать на текстовые сообщения;
- хранить отдельную память для каждого чата;
- показывать `chatId` через `/whoami`;
- принимать voice и audio;
- скачивать аудио из Telegram;
- распознавать речь через OpenRouter STT;
- отправлять длинные ответы частями до 3900 символов.

## Как работает обработка сообщения

1. `Agent.process()` получает текст пользователя.
2. Сначала проверяются прямые команды: `/help`, `/tools`, память, калькулятор, погода, поиск и так далее.
3. Если команда относится к анализу проекта, запускается `tools.codebase`.
4. Если сообщение содержит URL и просьбу прочитать страницу, запускается `webReader`.
5. Если прямого правила нет, `routerAgent` пытается выбрать инструмент.
6. Если инструмент выбран, результат передается в `resultAnalyzer`.
7. Если инструмент не нужен, сообщение идет в обычный LLM-ответ через `askAi()`.

## Анализ результатов

Файл `src/resultAnalyzer.js` превращает сырой вывод инструментов в человеческий ответ.

Для `codebase` используется отдельный prompt Codebase Analyzer. Для остальных инструментов используется общий Ranker/Analyzer, который убирает мусор, выбирает главное и отвечает кратко на русском.

## Ограничения

- Большая часть AI-функций требует `OPENROUTER_API_KEY`.
- Интернет-поиск требует `TAVILY_API_KEY`.
- YouTube-поиск требует `YOUTUBE_API_KEY`.
- Telegram-режим требует `TELEGRAM_BOT_TOKEN`.
- Если `DATABASE_URL` не задана, память хранится локально в `memory.json`.
- `Codebase Analyzer` анализирует snapshot с лимитами, а не бесконечный полный проект.
- Роутер инструментов зависит от качества ответа LLM, поэтому для важных функций есть прямые команды.
